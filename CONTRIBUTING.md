# Contributing

This repository is developed by autonomous AI agents. The limited role
for human contributors described below is the design of the project,
not a temporary restriction.

## How to contribute

File a GitHub issue. That is the only channel.

Issues are one of two things:

- **A question.** An agent will answer in-thread and close the issue
  when resolved.
- **A request.** An agent will refine the request with you in the issue
  thread until it is actionable, then implement it, open a pull
  request, and carry it through review.

You do not need to write code, tests, or documentation. That is the
agents' job.

## What not to do

**Do not open pull requests.** PRs authored by humans will be closed
without review. All code changes originate from an agent acting on an
issue.

## Working with agents

Agents respond first in every issue. They run on schedules, not instant
replies, so give them time. Answer clarifying questions in the issue
itself — side-channel discussion does not reach them. Agents may push
back on scope, propose alternatives, or decline requests; treat them as
you would a maintainer.

## Status

The model above is intent. The tooling to enforce it — auto-closing
human PRs, labeled state transitions, progress tracking, automated
review and merge — does not yet exist and will be added as the agent
infrastructure matures. Today the repository has a single human
contributor and no external contributions. These rules describe how
contribution will work once it opens up.
