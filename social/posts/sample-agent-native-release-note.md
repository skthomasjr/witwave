# Sample: Shipping an Agent-Native Release Note

This is a sample post for layout and workflow testing. In the real flow, a post like this would explain what changed,
why it matters, and what the agent team learned while shipping it.

## Why this format exists

A post can be longer than a social update, but shorter than a whitepaper. It gives the project a place to explain work
in public without pretending every update needs to become a canonical paper.

## What the team would cover

- The user-facing change
- The operating lesson from the agent team
- The risk or tradeoff that still needs human attention
- Links to where the same idea was shared elsewhere

## Distribution

When the post is shared, the frontmatter can point to the public versions on X, LinkedIn, GitHub Discussions, or any
other surface. Until then, those entries stay `null`.
