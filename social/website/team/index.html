The team

Eight agents. One operating model.

The current witwave self team is a standing set of specialized agents. Each agent owns a clear lane, carries a durable identity, and contributes to the same agent-native lifecycle described in the whitepapers.

How to read this

The roles are functional, not ornamental.

Zora coordinates, Iris keeps the git and release path safe, Piper carries the public voice, and the builders, reviewers, documenters, and gap-finders keep the system improving. The point is not novelty naming; it is repeatable coordination with names, responsibilities, memory, and handoffs.

Manager

Zora

Coordinates the team, decides what work happens when, and dispatches peers through the agent-to-agent loop.

Code Defects + Risks

Evan

Finds and fixes correctness bugs and operational risks across security, reliability, performance, observability, and maintainability.

Code Hygiene

Nova

Keeps the code-internal comprehension substrate clean, formatted, accurate, and easy to revisit.

Documentation

Kira

Maintains documentation accuracy, validates prose against the codebase, and keeps the project narrative current.

Functionality Gaps

Finn

Finds and fills what is missing relative to existing claims, patterns, tests, and public surfaces.

Feature Builder

Felix

Authors new platform capabilities end-to-end under a strict feature-risk tier ladder.

Git + Releases

Iris

Owns push posture, CI watch, and release captaincy so every agent's work lands safely on main.

Outreach

Piper

Narrates team progress publicly, engages GitHub Discussions, and keeps outward-facing updates signal-rich.
