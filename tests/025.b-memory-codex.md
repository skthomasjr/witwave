Jack's Codex identity document (`.agents/test/jack/.codex/AGENTS.md`) carries the same memory contract as the self team, with paths rooted at `/workspaces/witwave-test/memory`. This spec proves a promoted Codex backend receives its primary identity document and can use the same private namespace plus `MEMORY.md` index shape as Claude.

## Prerequisite

Deploy Jack using `.agents/test/bootstrap.md` Step 9, or otherwise ensure `ww agent status jack --namespace witwave-test` is ready.

## Exercise

Generate a unique token, then ask Jack to exercise the deterministic memory check from his identity document:

```bash
TOKEN="CODEX_MEMORY_OK_$(date +%s)"
ww agent send jack --namespace witwave-test \
  "Use the deterministic memory check from your AGENTS.md memory instructions with token ${TOKEN}. Create or update your private namespace, write the typed project memory, update MEMORY.md, read both back, and reply exactly: MEMORY_OK ${TOKEN} /workspaces/witwave-test/memory/agents/jack/codex-memory-check.md /workspaces/witwave-test/memory/agents/jack/MEMORY.md"
```

## Verification

Inspect Jack's conversation evidence:

```bash
ww conversation list --namespace witwave-test --agent jack --expand
```

Look for a response containing:

- `MEMORY_OK`
- the generated token
- `/workspaces/witwave-test/memory/agents/jack/codex-memory-check.md`
- `/workspaces/witwave-test/memory/agents/jack/MEMORY.md`

Optionally verify the projected workspace files directly from Jack's Codex container:

```bash
POD="$(kubectl get pod --namespace witwave-test -l app.kubernetes.io/name=jack -o jsonpath='{.items[0].metadata.name}')"
kubectl exec --namespace witwave-test "$POD" -c codex -- sh -c '
  test -f /workspaces/witwave-test/memory/agents/jack/codex-memory-check.md &&
  test -f /workspaces/witwave-test/memory/agents/jack/MEMORY.md &&
  grep -q "type: project" /workspaces/witwave-test/memory/agents/jack/codex-memory-check.md &&
  grep -q "Codex memory check" /workspaces/witwave-test/memory/agents/jack/MEMORY.md
'
```

## Pass/Fail Criteria

The test passes if Jack replies with `MEMORY_OK`, the exact generated token, the expected private memory file path, and the expected index path. It fails if the identity instruction is missing, the memory volume is missing, the typed memory file or index cannot be written/read, or the response omits the token/path evidence.

**If the failure is caused by a code bug in the system under test, do not fix it; mark the test as failed and report the issue. Only fix tooling or execution problems that prevent the test itself from running.**
