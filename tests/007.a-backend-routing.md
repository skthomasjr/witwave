Bob's `backend.yaml` routes all concerns (`a2a`, `heartbeat`, `job`, `task`, `trigger`, `continuation`) to the `codex` backend (`bob-a2-codex` at port 8091). This test verifies that traffic actually reaches the configured backend by inspecting the `agent` field in the shared conversation log.

All backends write to the shared conversation log at `.agents/test/bob/logs/conversation.jsonl`. Each entry includes an `"agent"` field identifying which backend wrote it (e.g., `"bob-a2-codex"` or `"bob-a2-claude"`). If routing is working correctly, entries for `JOB_OK` and `TRIGGER_OK` must have `"agent": "bob-a2-codex"`.

The test does not need to fire new requests — tests 001-006 have already exercised all these paths. Simply verify the log entries.

## Verification

Read `.agents/test/bob/logs/conversation.jsonl` (one JSON object per line).

Check that:
- At least one entry contains `JOB_OK` in its `text` field and has `"agent": "bob-a2-codex"`
- At least one entry contains `TRIGGER_OK` in its `text` field and has `"agent": "bob-a2-codex"`
- No entry containing `JOB_OK` in its `text` field has `"agent": "bob-a2-claude"`
- No entry containing `TRIGGER_OK` in its `text` field has `"agent": "bob-a2-claude"`

`HEARTBEAT_OK` may or may not be present; skip that check if absent.

## Pass/Fail Criteria

The test passes if:
1. At least one log entry with `JOB_OK` has `"agent": "bob-a2-codex"`.
2. At least one log entry with `TRIGGER_OK` has `"agent": "bob-a2-codex"`.
3. No log entry with `JOB_OK` has `"agent": "bob-a2-claude"`.
4. No log entry with `TRIGGER_OK` has `"agent": "bob-a2-claude"`.

The test fails if any of the above conditions are not met.

**If the failure is caused by a code bug in the system under test, do not fix it — mark the test as failed and report the issue. Only fix tooling or execution problems that prevent the test itself from running.**
