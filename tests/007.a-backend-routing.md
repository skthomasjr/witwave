Bob's default `backend.yaml` routes all concerns (`a2a`, `heartbeat`, `job`, `task`, `trigger`, `continuation`) to the `claude` backend. This test verifies that traffic reaches the configured backend by inspecting conversation evidence.

Use `ww conversation` as the source of truth for deployed smoke runs:

```bash
ww conversation list --namespace witwave-test --agent bob --expand
```

If routing is working correctly, entries for `JOB_OK` and `TRIGGER_OK` must have `agent` / backend identity `bob-claude`. Codex and Gemini entries should not appear in the default Bob smoke run.

The test does not need to fire new requests; tests 001-006 have already exercised these paths. Simply verify the log entries.

## Verification

Check that:

- At least one entry contains `JOB_OK` and is attributed to `bob-claude`.
- At least one entry contains `TRIGGER_OK` and is attributed to `bob-claude`.
- No entry containing `JOB_OK` is attributed to `bob-codex` or `bob-gemini`.
- No entry containing `TRIGGER_OK` is attributed to `bob-codex` or `bob-gemini`.

`HEARTBEAT_OK` may or may not be present; skip that check if absent.

## Pass/Fail Criteria

The test passes if all verification checks are true. It fails if any active smoke entry is routed to a backend other than `bob-claude`.

**If the failure is caused by a code bug in the system under test, do not fix it; mark the test as failed and report the issue. Only fix tooling or execution problems that prevent the test itself from running.**
