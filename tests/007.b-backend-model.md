Bob's `backend.yaml` routes all concerns to the `codex` agent with model `gpt-5.1-codex`. This test verifies both the agent and the model by inspecting the shared backend log after sending a fresh A2A request.

## Setup

Send a fresh A2A request to bob's nyx agent to produce a new trace entry:

```
curl -s -X POST http://localhost:8099/ \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"007b","method":"message/send","params":{"message":{"role":"user","messageId":"msg-007b","contextId":"routing-model-check","parts":[{"kind":"text","text":"Respond with MODEL_CHECK_OK."}]}}}'
```

Wait for the response to contain `MODEL_CHECK_OK` before proceeding.

## Verification

All backends write to the shared log at `.agents/test/bob/logs/conversation.jsonl`. Each entry includes an `"agent"` field.

**Agent check** — confirm the request was handled by the codex backend (not claude):

Read `.agents/test/bob/logs/conversation.jsonl` and find entries containing `MODEL_CHECK_OK`. Verify:
- At least one such entry has `"agent": "bob-a2-codex"`
- No such entry has `"agent": "bob-a2-claude"`

**Model check** — confirm the correct model was used:

Parse `.agents/test/bob/logs/trace.jsonl` (one JSON object per line). Find the entry whose `session_id` corresponds to the `routing-model-check` context. The `model` field must equal `gpt-5.1-codex`.

Since `routing-model-check` is not a UUID, the harness derives a deterministic UUID via `uuid5(NAMESPACE_URL, "routing-model-check")`. Compute that value and match it against the trace entries:

```
python3 -c "import uuid; print(uuid.uuid5(uuid.NAMESPACE_URL, 'routing-model-check'))"
```

Then grep the trace log for that session_id and verify the model field:

```
grep <computed-session-id> .agents/test/bob/logs/trace.jsonl
```

## Pass/Fail Criteria

The test passes if:
1. The A2A response contains `MODEL_CHECK_OK`.
2. At least one log entry with `MODEL_CHECK_OK` has `"agent": "bob-a2-codex"`.
3. No log entry with `MODEL_CHECK_OK` has `"agent": "bob-a2-claude"`.
4. The trace entry for the session has `"model": "gpt-5.1-codex"`.

The test fails if any of the above conditions are not met.
