Bob's default `backend.yaml` routes all concerns to the `claude` backend with model `claude-opus-4-7`. This test verifies both the backend and the model by sending a fresh A2A request and inspecting conversation evidence.

## Setup

Send a fresh A2A request to Bob's witwave agent:

```bash
curl -s -X POST http://localhost:8099/ \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"007b","method":"message/send","params":{"message":{"role":"user","messageId":"msg-007b","contextId":"routing-model-check","parts":[{"kind":"text","text":"Respond with MODEL_CHECK_OK."}]}}}'
```

Wait for the response to contain `MODEL_CHECK_OK` before proceeding.

## Verification

Inspect Bob's conversation log through the CLI:

```bash
ww conversation list --namespace witwave-test --agent bob --expand
```

Find entries containing `MODEL_CHECK_OK`. Verify:

- At least one such entry is attributed to `bob-claude`.
- No such entry is attributed to `bob-codex` or `bob-gemini`.
- The entry's `model` field is `claude-opus-4-7`.

## Pass/Fail Criteria

The test passes if:

1. The A2A response contains `MODEL_CHECK_OK`.
2. At least one conversation entry with `MODEL_CHECK_OK` is attributed to `bob-claude`.
3. No `MODEL_CHECK_OK` entry is attributed to `bob-codex` or `bob-gemini`.
4. The conversation entry model is `claude-opus-4-7`.

The test fails if any condition is not met.

**If the failure is caused by a code bug in the system under test, do not fix it; mark the test as failed and report the issue. Only fix tooling or execution problems that prevent the test itself from running.**
