HOOK EVENTS TOKEN (REQUIRED FOR BACKEND→HARNESS HOOK.DECISION POSTS)
--------------------------------------------------------------------
The harness exposes /internal/events/hook-decision on the metrics listener
(#924) and rejects every unauthenticated request. Backends POST hook.decision
events there via HARNESS_EVENTS_URL (auto-wired by this chart) and must
carry a matching bearer. The chart does NOT render this Secret for you —
the same token must land on BOTH the harness container AND every backend
container in the pod.

Create a shared Secret per agent and reference it via envFrom on both
containers (example wires `adam`):

  kubectl create secret generic {{ .Release.Name }}-adam-hook-events \
    --from-literal=HOOK_EVENTS_AUTH_TOKEN=$(openssl rand -hex 32) \
    -n {{ .Release.Namespace }}

Then in values.yaml for the agent add:

  envFrom:
    - secretRef:
        name: {{ .Release.Name }}-adam-hook-events
  # …and the SAME entry under each backend's envFrom.

Leaving HOOK_EVENTS_AUTH_TOKEN unset is a supported posture (hook.decision
POSTs are disabled) but the in-process OTel span emission still fires.
See #1590.

If using git-sync with inline credentials, the chart renders the Secret for you.
For an `existingSecret:`-backed gitSync credentials, create it yourself:
  kubectl create secret generic <my-gitsync-secret> \
    --from-literal=GITSYNC_USERNAME=<user> \
    --from-literal=GITSYNC_PASSWORD=<token> \
    -n {{ .Release.Namespace }}

If pulling images from a private registry:
  kubectl create secret docker-registry ghcr-credentials \
    --docker-server=ghcr.io \
    --docker-username=<user> \
    --docker-password=<token> \
    -n {{ .Release.Namespace }}

RELIABILITY
-----------
{{- /* #1625: PodDisruptionBudget default flipped to enabled=true with
      minAvailable=1. Safe at replicas=1 — see values.yaml comment block. */}}
NOTE: Since chart #1625, `podDisruptionBudget.enabled` defaults to **true**
(`minAvailable: 1`). This blocks `kubectl drain` on the agent's node until a
replacement pod is Ready elsewhere, eliminating the ~30s outage that previously
hit heartbeats/jobs/triggers on every node drain. Existing installs keep their
own `values.yaml` setting; only fresh installs (and `helm upgrade` runs that
don't pin the value) pick up the new default. Opt out with:
  helm upgrade {{ .Release.Name }} ... --set podDisruptionBudget.enabled=false

This is the default shape and is fine for dev/test. In production it means
every `kubectl drain`, node upgrade, or cluster-autoscaler scale-down causes
a brief outage: the harness's scheduled jobs, heartbeats, and triggers miss
fire windows, and in-flight A2A calls return 5xx until the replacement pod
is Ready (typically ~30s).

Recommended mitigation (safe at replicas=1 — see charts/witwave/README.md#reliability):
  helm upgrade {{ .Release.Name }} ... --set podDisruptionBudget.enabled=true

This enables a PodDisruptionBudget with `minAvailable: 1`, which blocks the
drain until a replacement pod is Ready on another node instead of dropping
the pod immediately. See #559 for background.
